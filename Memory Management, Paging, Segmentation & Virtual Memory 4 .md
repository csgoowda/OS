# Memory Management, Paging, Segmentation & Virtual Memory

⚠️ This is one of the most important OS topics.

Interviewers commonly ask:

* What is Virtual Memory?
* Paging vs Segmentation?
* What is Fragmentation?
* What is Thrashing?

To understand these topics, first understand **why memory management exists**.

---

# Why Do We Need Memory Management?

Suppose your laptop has:

```text
RAM = 8 GB
```

You open:

* Chrome → 2 GB
* VS Code → 1 GB
* Spotify → 500 MB
* Android Studio → 3 GB

Total:

```text
6.5 GB
```

Now open more applications.

RAM becomes full.

Questions:

* Which process gets memory?
* How much memory?
* What if RAM becomes full?

The OS handles all this through Memory Management.

---

# What is Memory Management?

Memory Management is the function of the OS that manages RAM efficiently.

Responsibilities:

* Allocate memory
* Deallocate memory
* Track memory usage
* Protect memory

---

# Memory Allocation

Suppose RAM:

```text
8 GB
```

Applications:

```text
Chrome = 2 GB
VS Code = 1 GB
Spotify = 500 MB
```

OS allocates memory blocks to each process.

---

# Problem Without Memory Management

Suppose:

Process A accesses memory of Process B.

Result:

```text
Data corruption
Security issue
```

Memory management prevents this.

---

# Logical Address vs Physical Address

Very important.

---

## Logical Address

Generated by CPU.

Example:

```text
Address = 500
```

Process thinks memory starts from zero.

---

## Physical Address

Actual location in RAM.

Example:

```text
Address = 25000
```

---

### Interview Answer

> Logical addresses are generated by the CPU, while physical addresses are actual locations in RAM.

---

# What is Paging?

Paging is a memory management technique that divides memory into fixed-size blocks.

---

# Why Paging?

Suppose process needs:

```text
10 MB
```

Available memory:

```text
4 MB
3 MB
3 MB
```

Memory is scattered.

Without paging:

Process cannot fit.

With paging:

Process is divided into smaller pieces.

Problem solved.

---

# Pages and Frames

### Page

Fixed-size block of process memory.

---

### Frame

Fixed-size block of physical memory.

---

Example:

```text
Process
--------
Page 1
Page 2
Page 3
```

RAM

```text
Frame 5
Frame 1
Frame 8
```

Pages can be stored anywhere.

---

# Paging Diagram

```text
Process

Page1 → Frame5
Page2 → Frame1
Page3 → Frame8
```

Not necessarily contiguous.

---

# Advantages of Paging

### No External Fragmentation

Big advantage.

---

### Efficient Memory Usage

Memory can be allocated in small chunks.

---

# Disadvantages of Paging

### Internal Fragmentation

Unused space inside a page.

---

### Page Table Overhead

Need extra memory for page tables.

---

# What is a Page Table?

Page table stores mapping between:

```text
Page Number
↓
Frame Number
```

Example:

| Page | Frame |
| ---- | ----- |
| 0    | 5     |
| 1    | 1     |
| 2    | 8     |

Used for address translation.

---

# What is Segmentation?

Segmentation divides memory based on logical units.

---

Example Program

```text
Code
Data
Stack
Heap
```

Each segment has different size.

---

# Segment Example

```text
Segment 1 → Code
Segment 2 → Data
Segment 3 → Stack
Segment 4 → Heap
```

---

# Paging vs Segmentation

Most Asked Question.

| Paging                    | Segmentation                    |
| ------------------------- | ------------------------------- |
| Fixed size blocks         | Variable size blocks            |
| Physical view             | Logical view                    |
| No external fragmentation | External fragmentation possible |
| Faster allocation         | More meaningful                 |

---

# Easy Memory Trick

Paging:

```text
All blocks same size
```

Segmentation:

```text
Different sizes
```

---

# Fragmentation

When memory gets wasted.

Two types:

1. Internal Fragmentation
2. External Fragmentation

---

# Internal Fragmentation

Unused space inside allocated memory.

---

Example

Memory block:

```text
100 KB
```

Process needs:

```text
80 KB
```

Unused:

```text
20 KB
```

Wasted inside allocated block.

---

This is Internal Fragmentation.

---

# External Fragmentation

Free memory exists but is scattered.

---

Example

```text
10 KB Free
20 KB Free
15 KB Free
```

Need:

```text
40 KB
```

Total memory:

```text
45 KB
```

Available.

But not contiguous.

Cannot allocate.

---

This is External Fragmentation.

---

# Internal vs External Fragmentation

| Internal           | External                   |
| ------------------ | -------------------------- |
| Waste inside block | Waste outside block        |
| Common in paging   | Common in segmentation     |
| Hard to utilize    | Can be fixed by compaction |

---

# What is Virtual Memory?

One of the most important OS concepts.

---

# Problem

Suppose:

```text
RAM = 8 GB
```

Program requires:

```text
12 GB
```

How can it run?

---

Answer:

Virtual Memory.

---

# Definition

Virtual memory is a technique that uses secondary storage (disk) as an extension of RAM.

---

# Working

When RAM becomes full:

OS moves less-used pages to disk.

This area is called:

```text
Swap Space
```

or

```text
Page File
```

---

# Example

RAM:

```text
8 GB
```

Program:

```text
12 GB
```

Extra:

```text
4 GB
```

Stored on disk.

---

# Real-Life Example

Study Table:

```text
RAM
```

Bookshelf:

```text
Disk
```

Frequently used books stay on table.

Others remain on shelf.

---

# Advantages of Virtual Memory

### Run Large Programs

Program larger than RAM can run.

---

### Better Multitasking

More applications can run simultaneously.

---

### Efficient Memory Usage

Only required pages loaded.

---

# Disadvantages

### Disk is Slower

Accessing disk is slower than RAM.

---

### Excessive Swapping Causes Thrashing

---

# What is Thrashing?

Extremely important interview topic.

---

# Definition

Thrashing occurs when the OS spends more time swapping pages than executing processes.

---

# Example

RAM:

```text
4 GB
```

Applications require:

```text
20 GB
```

OS continuously:

```text
RAM → Disk
Disk → RAM
RAM → Disk
Disk → RAM
```

CPU waits.

Performance drops.

---

# Symptoms of Thrashing

* System becomes slow
* High disk activity
* Low CPU utilization

---

# Solution

* Increase RAM
* Reduce multiprogramming
* Better page replacement algorithms

---

# Page Fault

Another common question.

---

# What is Page Fault?

Occurs when a required page is not present in RAM.

---

Example

Process needs:

```text
Page 5
```

RAM does not contain Page 5.

OS loads it from disk.

This event is called:

```text
Page Fault
```
